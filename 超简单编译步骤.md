# 超简单编译步骤（小白专用）

## 🎯 目标
生成一个 `otc-notification-listener.aar` 文件

---

## 📝 步骤 1：打开 Android Studio

1. 在应用程序中找到 `Android Studio`
2. 双击打开
3. 等待启动完成（可能需要30秒-1分钟）

---

## 📝 步骤 2：导入项目

1. 看到欢迎界面后，点击 **`Open`** 按钮
   - 如果没有看到欢迎界面，点击顶部菜单 `File` → `Open`

2. 在弹出的文件选择器中，找到并选择这个文件夹：
   ```
   /Users/xuelinggao/Documents/develop/work/otc-app/nativeplugins/otc-notification-listener/android
   ```
   
3. 点击 **`Open`** 按钮

4. 等待项目加载
   - 底部会显示 "Gradle Build Running..."
   - 第一次会很慢（可能5-10分钟），因为要下载依赖
   - 看到进度条走完就好了

⚠️ **如果卡住了**：
- 不要关闭窗口
- 等待即可
- 第一次一定会很慢，这是正常的

---

## 📝 步骤 3：等待同步完成

什么时候算完成？

✅ 底部状态栏不再显示 "Gradle Build Running"
✅ 底部显示 "Gradle build finished"
✅ 或者底部显示 "Build: Successful"

⚠️ **如果出现错误**：
- 看底部是否有红色错误信息
- 截图发给我，我帮你解决

---

## 📝 步骤 4：开始编译

### 方法 A：使用 Gradle 面板（推荐）

1. **找到右侧的 Gradle 图标**
   - 在 Android Studio 右侧边栏
   - 是一个大象图标 🐘
   - 点击它

2. **展开任务列表**
   - 点击 `android` 前面的小三角 ▶
   - 展开后，再点击 `Tasks` 前面的小三角 ▶
   - 再点击 `build` 前面的小三角 ▶

3. **双击 assembleRelease**
   - 在 `build` 下面找到 `assembleRelease`
   - 双击它
   - 会开始编译

4. **等待编译完成**
   - 底部会显示进度
   - 通常 1-2 分钟
   - 看到 "BUILD SUCCESSFUL" 就成功了！

### 方法 B：使用终端（备选）

如果找不到 Gradle 面板，用这个方法：

1. **打开终端**
   - 点击 Android Studio 底部的 `Terminal` 标签
   - 会打开一个命令行窗口

2. **输入命令**
   ```bash
   ./gradlew assembleRelease
   ```

3. **按回车**
   - 等待编译完成
   - 看到 "BUILD SUCCESSFUL" 就成功了

---

## 📝 步骤 5：找到生成的文件

编译成功后：

1. **在 Android Studio 左侧**
   - 点击 `Project` 面板
   - 找到并展开 `build` 文件夹
   - 展开 `outputs` 文件夹
   - 展开 `aar` 文件夹
   - 会看到 `android-release.aar` 文件

2. **右键点击这个文件**
   - 选择 `Reveal in Finder`（在访达中显示）
   - 会打开文件所在的文件夹

---

## 📝 步骤 6：复制文件

1. **在打开的 Finder 窗口中**
   - 找到 `android-release.aar` 文件
   - 右键点击，选择 `复制`

2. **返回上级目录**
   - 点击 Finder 窗口顶部的路径
   - 往上走几级，直到看到 `android` 文件夹
   - 在 `android` 文件夹外面（和 `android` 文件夹同级）粘贴

3. **重命名文件**
   - 把粘贴的文件重命名为：`otc-notification-listener.aar`

最终位置应该是：
```
nativeplugins/otc-notification-listener/android/otc-notification-listener.aar
```

## ❓ 常见问题

### Q1: 同步一直卡住怎么办？

**解决方法**：
1. 等待 10 分钟
2. 如果还是卡住，点击顶部菜单 `File` → `Invalidate Caches / Restart`
3. 选择 `Invalidate and Restart`
4. 重新打开项目

### Q2: 提示 SDK 不存在？

**解决方法**：
1. 点击错误提示中的 `Install SDK`
2. 等待安装完成
3. 重新编译

### Q3: 编译失败，显示红色错误？

**解决方法**：
1. 截图完整的错误信息
2. 发给我
3. 我帮你分析
