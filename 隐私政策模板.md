#### 1. 通知使用权（Notification Listener Service）

**权限名称**：通知使用权

- [示例] 监控特定应用的消息提醒

#### 2. 数据收集范围

当您授权我们使用通知监听权限后，我们可能会收集以下信息：

- 应用包名（用于识别通知来源）
- 通知标题
- 通知内容
- 通知时间

#### 3. 数据使用方式

我们承诺：

- ✅ 仅在您明确授权后使用此权限
- ✅ 仅用于提供应用核心功能，不会用于其他用途
- ✅ 不会将通知数据上传到服务器（除非功能必需且已告知）
- ✅ 不会与第三方共享您的通知数据
- ✅ 您可以随时在系统设置中关闭此权限

#### 4. 如何管理权限

您可以通过以下方式管理通知使用权：

**开启权限**：
1. 打开系统"设置"
2. 进入"通知"或"应用管理"
3. 找到"通知使用权"或"通知访问"
4. 找到本应用并开启权限

**关闭权限**：
1. 按照上述步骤进入通知使用权设置
2. 关闭本应用的通知使用权
3. 注意：关闭后，相关功能将无法使用

#### 5. 数据安全

我们采取以下措施保护您的通知数据：

- 🔒 本地存储加密
- 🔒 数据访问权限控制
- 🔒 定期清理过期数据

#### 6. 用户权利

您拥有以下权利：

- 📋 查询我们收集的通知数据
- 🗑️ 删除我们存储的通知数据
- ⏸️ 暂停通知监听服务
- 🚫 拒绝或撤回授权

```javascript

  title: '通知使用权说明',

  confirmText: '我已了解',
  success: (res) => {
    if (res.confirm) {
      // 引导用户开启权限
      notificationModule.openSettings();
    }
  }
});
```

## 重要提醒

### ⚠️ 法律法规

- 遵守《中华人民共和国网络安全法》
- 遵守《中华人民共和国个人信息保护法》
- 遵守《App违法违规收集使用个人信息行为认定方法》
- 遵守各应用商店的审核规范

### ⚠️ 禁止行为

- ❌ 未经用户同意收集通知数据
- ❌ 将通知数据用于未告知的用途
- ❌ 与第三方共享用户的通知内容
- ❌ 上传用户的敏感通知信息（如验证码、密码等）
- ❌ 监控用户的私密通信内容

### ⚠️ 应用商店审核

部分应用商店对通知监听权限有严格审核：

- 需要在应用说明中明确功能用途
- 可能需要提供视频演示
- 需要提供完整的隐私政策
- 可能需要提供用户协议
